# PulseGrid System

A Java-based Object-Oriented Programming project designed to demonstrate core inheritance concepts through a dynamic role-based system.

 Overview
PulseGrid models a system where different entities (users, active users, and admins) interact within a structured hierarchy.  
The project focuses on behavior and structure rather than static data, making inheritance visible through method execution and system flow.


 Concepts

- Class Inheritance (`extends`)
- Superclass vs Subclass
- Single Inheritance
- Multilevel Inheritance
- Hierarchical Inheritance
- Method Overriding
- this Keyword
- super Keyword (methods & constructors)
- super() Constructor Calls
- Protected Access Modifier
- Method Hiding (static methods)
- final Class
- final Method
- Package Structure & Visibility
- Implicit super() behavior
- Java Multiple Inheritance Restriction


Features

- Clear inheritance hierarchy with real behavioral flow  
- Demonstrates method overriding and method hiding  
- Uses super and this effectively  
- Includes a final class and method restriction  
- Organized using Java packages  

Purpose

This project was developed as part of an Object-Oriented Programming assignment to demonstrate understanding of inheritance and related Java concepts through a structured and functional system.
